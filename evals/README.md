# Sifta 端到端质量基线

回答"整体几分、短板在哪"，用可复现的方式而不是单点验证拼分。

## 组成

- `briefs.json` —— 四画像各一个真实招聘 brief（工程 / 学术 / GTM / 创始人），默认中国优先。
- `rubric.md` —— 跨画像统一的 7 维评分表（每维 0/1/2，总 14，换算 10 分制）。
- 结果落 `results/<日期>/<brief-id>/`：`report.md`（宿主 Agent 产出的人选报告）+ `grade.json`（评分员按 rubric 打分）。

## 怎么跑一轮 baseline

1. **寻访**：每个 brief 派一个宿主 Agent，提示它读取对应 skill 的 `SKILL.md` 并**严格按 skill 执行完整寻访**，完成 brief，产出人选报告（每人：真名/身份互链、证据链接、可达信号、地域）。它可以跑 skill 里的脚本、联网核验。评分方向不告诉它（避免应试）。
   - 网络：HF 脚本直连不通时用自有代理（`HTTPS_PROXY`）；OpenAlex 学术脚本要直连；GTM 走 sifta-app 的 Exa 入口（key 在 DB/.env）。
2. **评分**：另派一个评分员 Agent，只给它 `rubric.md` + brief + 产出报告，逐维打分 + 写短板。评分员不参与寻访。
3. **汇总**：四画像分 + 均值 = 端到端整体分；把每维非满分的短板汇成"短板地图"，作为下一轮迭代的输入。

## 纪律

- 评分只认可核验证据（报告得给能点开的链接/署名）。
- 隐私是硬门（私人联系方式 / 推断求职意愿 → 维度 6 直接 0）。
- 诚实缺口优于凑数。
- 这是过程质量回归工具：改 skill 后重跑，看四画像分和短板地图有没有真的变好。
