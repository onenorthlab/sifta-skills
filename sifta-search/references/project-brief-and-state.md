# 项目简报和候选人状态

本文件定义 Sifta 候选人寻访的输入前置检查和 `lead -> candidate` 升级门槛。目标是少问问题，
但不把来源地图线索、论文作者、仓库 owner、公司成员或实验室网页直接包装成候选人。

## 1. 项目简报门槛

先把用户输入压缩成项目简报。用户不需要提供渠道、来源或准确头衔；先理解他要补什么能力，再由 Agent 派生路线和来源。缺项分为两类：

| 字段 | 用途 | 缺失时 |
| --- | --- | --- |
| `capabilityShape` | 用户要找的人能解决什么问题、做过什么结果、拥有哪些经历信号 | 无法判断任何能力方向时硬停止，只问一个短问题 |
| `inferredRoute` | 由能力画像派生工程、产品/GTM、研究、创始人/运营型负责人、候选人档案或触达文案路线 | 不要求用户提供；可推断就写假设 |
| `evidencePriority` | 内部选择 GitHub、LinkedIn、学术来源栈、公开个人资料或已核验证据 | 不主动问用户；用默认来源地图推进 |
| `mustHave` | 技能、论文、开源、产品、公司、市场、职级或项目阶段 | 不阻塞；写假设和适配证明要求 |
| `avoid` | 排除公司、人群、风险、不可说内容 | 不阻塞；涉及隐私、自动发送或非公开数据时硬停止 |
| `quantity` | 候选人数或名单深度 | 默认 3-5；用户要求最强时先给 1 个最强 |
| `geoBias` | 候选人地域、市场、语言或人才池偏置 | 用户未指定时默认 `中国/中文生态相关人才池优先`；不做族裔推断 |
| `sourceConstraints` | 只有用户主动指定 GitHub / LinkedIn / X / 学术 / 原生搜索 / CLI 时才记录 | 必须保留；重试和回退不得静默换来源 |

默认地域规则：

- 用户没有说明地区、远程、全球或海外市场时，写入 `geoBias=中国/中文生态相关人才池优先`，并放入“假设”。
- 默认地域/市场不是族裔、国籍或姓名判断；只接受公开职业资料里的中国大陆、港澳台、中文教育/工作/社区、中国市场或中国相关机构/公司信号。
- 默认地域会影响来源地图、查询词和后续核验：优先找中国/中文生态相关公开职业信号；不要凭姓名、照片、外貌、口音、族裔、国籍或猜测判断。
- 默认地域不是严格 `filter.locations`；不要因此删除来源地图里的强全球证据。地域未知或不在默认范围的人选必须写覆盖风险和下一步核验。
- 默认地域是候选人升级门槛：如果用户没有明确放宽为全球人才池，缺公开中国/中文生态相关职业信号的人不能进入候选人或强线索分桶，只能进入来源地图、产业标杆、推荐人或待核验线索。
- 只有用户明确说“中国/上海/北京/海外/北美/远程/全球”等硬约束时，才把地域写成严格过滤条件。

硬停止只用于关键边界：

- 不清楚是否招聘还是公司/市场研究。
- 只说“AI 人才/强人”且能力画像不可推断。
- 已知候选人深挖缺少可消歧个人资料或公司/地点线索。
- 用户要求私人联系方式、手机号、自动发送或非公开数据。
- 用户要求销售/BD/合作线索、私人联系方式或批量外联，且没有招聘候选人目标。
- 用户明确要求马上执行付费连接器，但未授权或未配置。

硬停止回复只包含一个短问题和一句原因，不输出流程、CLI 命令、来源地图或候选人表。

## 2. 线索状态机

候选人必须按状态升级。不要跳过中间门槛。

```text
source-map lead
  -> identity checked
  -> evidence graded
  -> candidate bucketed
  -> outreach-ready / rejected
```

| 状态 | 可进入来源 | 升级条件 | 不能做 |
| --- | --- | --- | --- |
| source-map lead | 论文、仓库、公司、实验室、导师、共同作者、竞赛、项目、数据集 | 找到个人资料、GitHub、LinkedIn、个人主页、实验室简介或用户明确提供的个人资料 | 不能输出为候选人推荐 |
| identity checked | 个人资料、GitHub、LinkedIn、个人主页、Scholar 个人资料、实验室简介 | 同人置信度至少中等，或明确标为待核验 | 不能跨渠道硬合并 |
| evidence graded | 个人资料 + 与用户要求相关的公开证据 | 给出 A/B/C/Reject 和风险 / 弱项 | 不能用头衔关键词或 star/fork 数替代证据 |
| candidate bucketed | 已分全职候选、顾问/推荐人、产业标杆、待核验、排除 | 适配证明包 完整；若使用默认中国/中文生态人才池，还要有公开相关职业信号，或用户明确放宽为全球人才池 | 不能把顾问/标杆包装成普通全职候选；不能用姓名/照片/族裔猜测补地域信号 |
| outreach-ready | 已核验证据 + 项目/岗位 + 渠道 + 不可说内容边界 | 人工确认后才能发送 | 不能自动发送或猜联系方式 |
| rejected | 身份冲突、非公开数据、无关、证据无法追溯 | 写明拒绝或排除原因 | 不能为了凑数保留 |

## 3. 来源地图最小字段

每条来源地图线索至少包含：

| 字段 | 说明 |
| --- | --- |
| `lead` | 论文、仓库、公司、实验室、项目、作者、贡献者、候选人线索名 |
| `sourceFamily` | GitHub / LinkedIn / academic / company / lab / project / user-provided |
| `whyRelevant` | 为什么与用户需求相关 |
| `conversionBlocker` | 还缺什么才可进入候选表，例如身份、个人资料、职业阶段、实现证据 |
| `nextVerification` | 下一步核验动作，例如找个人主页、GitHub 个人资料、LinkedIn、实验室简介、仓库提交 |

## 4. 证据等级

| 等级 | 口径 | 可输出为候选人吗 |
| --- | --- | --- |
| A | 个人资料与目标证据直接匹配，来源可追溯，身份/适配/证据都清楚 | 可以 |
| B | 单一强个人资料或强来源地图证据，缺交叉验证 | 可以作为待复核候选，必须写覆盖风险 |
| C | 仓库 owner、论文作者、公司/实验室成员、关键词命中、通用列表 | 只能作为来源地图线索或待核验 |
| Reject | 身份冲突、非公开信息、隐私请求、无关、证据无法追溯 | 不进入候选表 |

硬边界优先于分数：私人联系方式、自动发送、无个人资料、身份冲突、非公开数据、
销售线索/BD/合作/KOL/公司研究误路由，直接进入硬停止或排除。

默认地域/市场边界也优先于分数：全球开源贡献、论文或公司职位很强，但没有公开中国/中文生态相关职业信号时，不能因为要凑数而进入候选人或强线索分桶。可以输出为 `source-map lead`、`产业标杆`、`推荐人` 或 `地域未核验线索`，并说明需要用户批准放宽或下一轮定向核验。

## 5. CLI / 宿主原生能力边界

| 场景 | 默认执行面 | CLI 何时使用 |
| --- | --- | --- |
| GitHub 工程小批量 | 内置 `small-batch-github.mjs`，底层使用 GitHub REST；优先读 `GH_TOKEN` / `GITHUB_TOKEN`，没有 token 时匿名低额度请求 | 目标数 >3、辅助脚本不适用或用户批准下一轮时才转宿主原生能力；不要只因 GitHub token 改走 Sifta CLI |
| GitHub 工程广泛搜索 | 宿主 GitHub 搜索 / GitHub MCP / `gh` / 用户 `GH_TOKEN` 或 `GITHUB_TOKEN` | 只有需要 Sifta 统一 JSON、调用轨迹、反馈闭环、回归验证或 direct API 合同时才用 CLI |
| LinkedIn 产品/GTM 小批量 | 内置 `small-batch-product-gtm.mjs` 作为 Sifta CLI 包装 | 1-3 人单轮；连接器不可用时停在来源方案 |
| LinkedIn 产品/GTM 广泛搜索 | Sifta CLI/API | 需要更大名单或公司地图辅助搜索 |
| 学术图谱 | 宿主学术 / 网页来源地图 | 需要研究轨迹、结构化候选人或人工反馈 |
| 候选人档案 | 宿主个人资料阅读 | 已知个人资料需要统一公开证据结构 |
| 人工反馈 | `sifta-cli find-people --feedback` | 需要把人工反馈转成按来源拆分的下一轮请求 |

CLI/API 不承担规划、长期记忆、通用网页搜索、ATS 写操作、自动外联或 provider key 管理。

小批量辅助脚本完成、失败或返回 0 人时，本轮不继续追加网页搜索、浏览器、`gh`、第二次
`find-people` 或换源。最终答复只写用户报告，不输出停止标记、脚本名、命令、参数或运行过程。
GitHub 额度或认证失败时，下一步是在宿主环境配置 `GH_TOKEN` / `GITHUB_TOKEN`、GitHub MCP
或 `gh auth` 后重跑；不要改走 Sifta CLI。后续个人资料核验只能作为用户批准后的下一轮动作。
