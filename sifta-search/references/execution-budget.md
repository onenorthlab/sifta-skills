# 执行边界和预算

本文件约束 Agent 在一次用户请求中的工具使用范围，避免小批量候选人寻访变成无边界搜索。

## 1. 默认预算

| 场景 | 推荐边界 |
| --- | --- |
| 先计划 | 只输出计划 / 来源地图 / 硬停止；不调用 CLI、网页搜索、浏览器或 `gh` |
| GitHub / 工程 1-3 个强线索 | 优先用场景 Skill 的小批量辅助脚本或宿主 GitHub 工具；不要同轮反复换源 |
| 产品/GTM 1-3 个强线索 | 优先用 Sifta CLI/API；连接器不可用时停在失败说明、覆盖风险和下一步，不用网页/浏览器/原生搜索替换候选人 |
| 学术图谱 | 先做来源地图；不要把论文作者直接升级为候选人 |
| 候选人档案 | 只读公开个人资料；由宿主 Agent 按档案结构整理证据 |
| 隐私硬停止 | 不搜索；说明边界并问一个必要澄清 |

## 2. 停止条件

满足任一条件时，本轮停止扩展并交付当前结果、覆盖风险、停止条件、执行合同和下一步：

- 用户只要 1-3 个强线索，已有可解释的候选或来源地图线索。
- 辅助脚本或 CLI 返回了明确的 provider/auth/结构/warnings 信息。
- 当前结果只有弱线索，继续搜索也无法补足个人资料证据。
- 需要更大范围扩展、跨来源二轮或人工反馈后再继续。
- 涉及私人联系方式、自动发送、ATS/CRM 写入或未授权付费连接器。

不要为了让列表变长而连续调用网页搜索、浏览器、`gh`、`sifta-cli find-people`
或多个辅助脚本。下一轮扩展应在用户确认后执行。

辅助脚本完成、失败、认证不可用或返回 0 人时，本轮仍然停止；最终答复只写用户报告：
项目简报、来源地图、候选人分桶、待核验线索、适配证明包、覆盖风险、停止条件、执行合同和下一步。不要把停止标记、脚本名、命令、
参数、运行过程、events、timing 或 `target-count` 写给用户。不要临时改用网页搜索、
浏览器、Exa、公司页或原生搜索补候选人。
小批量 helper 输出 `STOP_AFTER_HELPER=true` 时必须停止，不得继续换源或追加搜索。

## 3. 质量边界

执行路径稳定不等于候选质量通过。最终输出仍要分开说明：

| 维度 | 要求 |
| --- | --- |
| 相关性 | 候选人主要经历能回答用户能力画像 |
| 证据强度 | 证据可追溯到公开个人资料、仓库、论文、项目页或职业材料 |
| 来源可靠性 | 来源与候选人身份、职能和贡献匹配 |
| 缺口诚实度 | 明确写缺口，不推断求职意愿、私人联系方式、薪资、签证或搬迁意愿 |

弱结果只能作为来源地图线索、待核验线索或覆盖风险，不能包装成强推荐。
