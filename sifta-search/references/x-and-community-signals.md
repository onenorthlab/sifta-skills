# X 和社区公开信号

本文件定义什么时候使用 X 和中文/开发者社区信号。社区信号只补公开职业证据，不能替代候选人身份核验。

## 1. 什么时候用

| 场景                                     | 是否默认用             | 原因                                    |
| ---------------------------------------- | ---------------------- | --------------------------------------- |
| 用户明确说 X、公开帖子、社区表达、影响力 | 可以                   | 用户授权了来源                          |
| DevRel、开发者增长、独立开发者、超级个体 | 可以作为辅助来源       | 公开表达和社区反馈能证明分发/开发者关系 |
| 工程强证据不足                           | 只作补充               | 不能替代 GitHub/作品证据                |
| 学术图谱                                 | 只作身份和公开表达补充 | 不能替代论文和个人资料                  |
| 默认普通候选人搜索                       | 不自动加入 X           | 避免扩大到社交/KOL/隐私边界             |

## 2. 可用来源

| 来源                   | 用法                                      | 边界                                                 |
| ---------------------- | ----------------------------------------- | ---------------------------------------------------- |
| X / Twitter            | 公开技术表达、产品发布、开发者社区互动    | 只有公开帖；不抓私信，不推断求职意愿                 |
| Hugging Face           | model/dataset/space、paper/code 线索      | 需要回到个人资料或组织身份核验                       |
| ModelScope / 魔搭      | 中文模型、应用、数据集、社区项目          | 适合中文生态找人来源；仍需个人资料                   |
| Gitee                  | 中文开源/企业开源补充                     | 代码证据需要和个人资料互证                           |
| 知乎 / 即刻 / 公众号   | 产品/技术表达、行业影响力、独立开发者线索 | 内容影响力不是候选人充分证据                         |
| B 站 / 视频号 / 活动页 | DevRel、社区、课程、公开演讲              | 只作公开作品和影响力补充                             |
| Discord/飞书/微信群    | 默认不用                                  | 可能涉及私域或非公开数据；除非用户提供公开链接且允许 |

## 3. 日期窗口

如果用户要求“最近”“近半年”“某段时间内”的 X 公开表达，使用 `sourceOptions.x.fromDate`
和 `sourceOptions.x.toDate` 控制 X Search 时间范围，格式为 `YYYY-MM-DD`。

日期窗口只说明搜索了这个时间段内的公开 X 内容；不要据此推断候选人当前雇主、求职意愿、
可联系性或近期一定仍活跃。

X 和 GitHub、LinkedIn、学术来源一样，默认都是 `中国/中文生态相关人才池优先`。X 查询、
项目简报和候选人升级门槛都要保留中国大陆、港澳台、中文教育/工作/社区、中国市场或中国相关
机构/公司信号；不得凭姓名、头像、语言风格、族裔或国籍猜测。

## 4. 升级规则

社区信号进入候选人前必须补齐：

1. 个人资料或主页。
2. 与用户需求相关的作品/项目/职业证据。
3. 公开中国/中文生态信号（默认地域未放宽时）。
4. 风险说明：内容是否过期、身份是否同一人、贡献是否本人完成。

## 5. 输出口径

- 公开表达强但缺职业资料：放“待核验线索”，写下一步找个人主页/LinkedIn/GitHub。
- 内容影响力强但没有作品证据：不要包装成工程/产品/GTM 候选。
- 用户要求私信、邮箱、手机号或批量外联：按硬停止；可以写草稿，但不找私人联系方式、不自动发送。
