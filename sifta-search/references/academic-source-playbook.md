# 学术来源执行手册

本文件给学术图谱场景提供可复用执行配方。用于 WAM/VLA、基础模型、研究工程、高潜博士、论文到候选人的路径。

## 1. 来源路径

| 路径               | 先找什么                                 | 再做什么                                   | 常见失败                       |
| ------------------ | ---------------------------------------- | ------------------------------------------ | ------------------------------ |
| paper-first        | 方向关键词、近两年论文、一作/共一作      | 找作者主页、lab bio、GitHub、official code | 只得到作者列表，无法证明可招聘 |
| lab-first          | 中国/中文生态相关实验室、PI、项目页      | 找学生、博士后、project collaborator       | PI 变成标杆，不是全职候选      |
| coauthor-graph     | 共同作者、引用、师承关系                 | 从头部作者扩展年轻共同作者                 | 噪声大，需消歧                 |
| competition-signal | Kaggle、比赛、benchmark、challenge       | 找个人主页和代码实现                       | 奖项不等于当前职业阶段         |
| graph-neighbor     | OpenAlex/Semantic topics、related papers | 找相邻方向作者和项目                       | 方向相关但人不匹配             |
| advisor-entry      | 导师/PI/产业科学家                       | 作为推荐人入口，向学生/合作者扩展          | 误当全职候选                   |

## 2. 每源查询和边界

| 来源                         | 查询方式                                                  | 候选升级条件                        |
| ---------------------------- | --------------------------------------------------------- | ----------------------------------- |
| OpenAlex                     | works/authors/institutions/topics，扩引用、共同作者和机构 | 只作图谱；必须补个人资料            |
| Semantic Scholar             | paper/author/citation/reference                           | 可与 OpenAlex 交叉验证；仍需消歧    |
| arXiv/OpenReview             | 近期预印本、会议投稿、review 场景                         | 论文作者只进找人来源                |
| Google Scholar               | 广泛召回、漏召回、作者主页入口                            | 不假设官方 API；只作浏览器/人工入口 |
| Papers with Code / HF Papers | official code、benchmark、dataset                         | 找实现者或 repo，再核验个人资料     |
| 实验室 / 项目 / 个人主页     | 成员、项目、个人身份、当前阶段                            | 候选人升级关键证据                  |
| GitHub / LinkedIn / X        | 个人资料、实现、职业阶段、公开表达                        | 可把来源线索转成候选人              |

## 3. 最小执行序列

```text
用户研究画像
  -> 论文/实验室/项目找人来源
  -> 至少两条来源路径
  -> 个人资料核验
  -> 贡献和职业阶段核验
  -> 分类：全职候选 / 顾问推荐人 / 产业标杆 / 待确认 / 排除
```

## 4. 输出要求

- 如果没有个人资料，不输出完整候选人分桶，只输出来源地图和下一步核验动作。
- 如果只找到 PI、导师或头部科学家，先放顾问/推荐人/产业标杆。
- 如果候选人缺中国/中文生态公开职业信号，默认只能作为来源地图、推荐人或待核验线索。
- 覆盖风险必须说明哪些学术来源没有查到、访问受限或只作 broad recall。

## 5. 判断准则（你来判，别指望脚本枚举）

召回 helper（`small-batch-academic.mjs`）只做确定性的事：调 OpenAlex 召回、去重、按客观数值（引用/发文/近年活跃）粗排、给出 country_code 地域事实、封顶 soft。它**不判**"这篇是不是综述、这个作者是不是同名合并、方向对不对、是不是中国生态"——这些是语义判断，正则/枚举表枚举不尽、跨语言就失效，必须由你读候选人的原始证据（论文标题、concepts、机构、引用、共同作者）自己判。helper 输出里的 `roughBand`（high/mid/low）只是机械信号强度粗排，**不是**权威判级；最终 tier 以下面准则为准。

### 5.1 研究证据强度（主轴）

先判"这个人的研究证据有多硬"，再谈地域偏好。强证据 = 在目标方向上有一手研究产出且近年活跃：

- **综述/评述是弱证据**：标题/摘要像 survey、review、overview、systematic review、benchmark study、tutorial 的论文，高引主要来自"综述效应"，不证明一手研究能力。若候选人的相关证据几乎全是综述，别因为引用高就当强候选，降到相邻线索。（判断看标题语义，不要套固定词表——中英文、各种变体都算。）
- **一手研究 + 一作/通讯**：在目标方向有一作/共一/通讯的近两年论文，是最硬的证据。
- **近年活跃**：近 1-2 年持续发文说明还在一线；只有多年前老论文的人可能已转行/退圈。

### 5.2 方向是否契合（硬前提）

产出量高但方向不沾，只能是相邻线索，不是候选。判断候选人的论文标题/concepts 是否落在用户要的研究方向上——**语义判断，跨中英文**（用户可能中文描述方向、论文是英文标题）。种子腿（`--seed` 从具名标杆论文扩展）召回的人方向由标杆保证，可直接信；泛关键词腿召回的人要自己核方向，别被通用词误召的高产者骗了。

### 5.3 疑似同名合并实体（消歧）

OpenAlex 作者消歧不完全，会把多个同名真人合并成一个高引实体。疑似信号（你结合候选人数据判断，不套死清单）：

- concepts 横跨彼此无关的重学科（如同时高权重出现医学 + 材料 + AI），像拼接实体；
- 发文量异常高（如年均 > 25 篇）却没有 ORCID/主页等身份交叉锚，单实体难核验；
- 同一"作者"机构/国别在不同论文间剧烈跳变。

命中疑似信号的实体先降为待核验线索，标注消歧风险，别让噪声实体凭高引冒充强候选。

### 5.4 地域偏好（次轴，档内加分，不跨档反超）

默认中国/中文生态优先，但这是**偏好轴**不是证据轴：弱证据 + 强地域，不能反超强证据 + 弱地域。

- `country_code` = CN/HK/TW 是硬地域事实（helper 已给 `geoStrong`）；
- 机构名/公司是否属中国生态（清华、智谱、深势、某 AI 创业公司…）由你判断，别指望 helper 有完整机构清单——它故意不枚举；
- 地域只在同等证据档内加分，帮你在强候选里优先中国生态的人。

### 5.5 可招聘性（学术共性盲点）

学术来源证明"研究能力"，不证明"可招聘"。在读博士、PI、已创业的超级个体都可能不可招或不该按全职候选处理。所有学术候选最高 soft（helper 已封顶），需再核验个人资料 + 当前职业阶段（学生/博后/工业界/自营）才谈推进；不推断求职意愿、薪资、relocation，不查私人联系方式。
