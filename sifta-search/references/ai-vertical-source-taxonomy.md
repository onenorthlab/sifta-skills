# AI 垂直找人来源

本文件把 AI 行业方向、人才角色和公开来源对应起来。用于复杂画像、创始人型/超级个体、独立开发者、跨渠道寻访和普通用户只说“找能做 X 的人”时的内部决策。

## 1. 使用原则

- 先判断行业方向，再判断角色族，最后选择来源。不要让渠道名替代用户真正要补的能力。
- 默认找中国/中文生态相关人才池；只用公开职业资料、公司/学校/社区/产品/论文/项目中的信号，不凭姓名、照片、口音、族裔或国籍推断。
- 每个正式候选人至少需要一个个人资料入口和一个与需求直接相关的证据。论文、公司、实验室、仓库、产品页和媒体报道先是找人来源线索。
- 对 founder、超级个体、PI、头部 maintainer 和明星创业者，先区分“可直接联系的人选”“顾问/推荐人”“产业标杆”“待确认线索”。

## 3. 角色族

| 角色                  | 先看什么                                          | 常见误判                                             | 输出分类                                    |
| --------------------- | ------------------------------------------------- | ---------------------------------------------------- | ------------------------------------------- |
| 工程师                | 代码、架构、repo、PR、package、技术文章           | star/fork、awesome list、组织仓库 owner 直接当强候选 | 全职候选 / 待确认 / 产业标杆                |
| 研究员 / 研究工程师   | 论文贡献、代码实现、实验室身份、项目或 benchmark  | 论文作者、导师、PI 直接当全职候选                    | 高潜候选 / 顾问推荐人 / 产业标杆 / 来源线索 |
| 产品经理              | 产品 ownership、平台/应用方向、用户问题、上线记录 | 公司是 AI 就默认候选人做 AI 产品                     | 产品候选 / 待确认                           |

| 创始人型 / operator   | 0 到 1、产品/技术/增长组合、团队和用户证据        | 只因 title 是 founder 就当可招募                     | 创始人级候选 / 顾问 / 标杆                  |

## 5. 输出边界

- 只找到公司、论文、仓库、产品或实验室时，放进“待确认线索”，写还差什么。
- 只找到公开表达但没有职业资料或作品证据时，最多是待确认线索。
- 强技术人但无中国/中文生态公开职业信号时，除非用户放宽全球范围，否则放找人来源、标杆或待确认。
- 触达前必须再走候选人档案或已核验证据；不要从找人来源线索直接写私信。
