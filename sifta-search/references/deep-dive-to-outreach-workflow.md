# 深挖到建联工作流

本文件串起候选人 shortlist、档案深挖、触达草稿和反馈二轮。用于用户问“能不能初筛后深挖、私信建联、继续找类似的人”时。

## 1. 状态流

```text
shortlist
  -> dossier
  -> outreach-ready
  -> human review
  -> feedback ingest
  -> next sourcing round
```

| 状态 | 输入 | 输出 | 禁止 |
| --- | --- | --- | --- |
| shortlist | 候选人表或来源地图 | 候选人分桶、待核验线索、覆盖风险 | 把来源线索直接当可联系候选人 |
| dossier | 已知个人资料或候选人 key | 身份、时间线、作品/论文/产品/GTM 成就、公开职业渠道、风险 | 猜私人邮箱、手机号、住址、家庭信息 |
| outreach-ready | 已核验证据 + 岗位/项目 + 渠道 | 私信/邮件/LinkedIn/引荐草稿和确认清单 | 自动发送、编造关系、承诺薪资/职级/签证 |
| human review | 用户反馈 | 喜欢/不喜欢、排除、扩展种子、下一轮约束 | 把长反馈塞进 GitHub query |
| next sourcing round | 反馈 JSON + 来源拆分 | 下一轮候选人或来源地图 | 静默换源、丢掉用户指定来源 |

## 2. 深挖要求

候选人档案必须回答：

- 是不是同一个人：跨资料证据和不确定点。
- 现在做什么：公开职业资料和时间线。
- 为什么相关：和用户需求最直接的 1-3 条证据。
- 还差什么：身份、贡献深度、当前阶段、地域/市场、是否适合全职。
- 公开职业联系方式：只包括个人主页公开 email、GitHub profile email、论文通讯邮箱或公开社交链接；没有就写没有看到。

## 3. 触达要求

触达文案只基于已核验证据：

1. 选 1-2 个最强个性化切入点。
2. 说明为什么这个项目和他的公开经历有关。
3. 保持短，不夸大，不承诺薪资、职级、融资或共同熟人。
4. 给出人工确认清单：人名、证据、渠道、不可说内容。

## 4. 反馈二轮

把用户反馈转成结构化约束：

| 用户反馈 | 下一轮处理 |
| --- | --- |
| `这个人更像顾问` | 加入排除/降级规则，向学生、前同事、共同作者或 team member 扩展 |
| `太学术，想要能落地` | 强化 GitHub/产品/公司项目证据，减少纯论文线索 |
| `方向对，但要中国市场` | 保留默认地域门槛，补中国/中文生态公开职业信号 |
| `不要大厂现任高管` | 排除或降级为产业标杆，找相邻公司早期员工 |
| `这几个都不错，找类似的` | 提取共同证据信号和来源族，按来源拆下一轮 |
