# Sifta 触达文案

用于已知候选人、上一轮结果或候选人档案之后的触达文案。目标是生成有证据支撑、可人工确认的招聘方 / 创始人 / 用人经理草稿，不做发送自动化，不抓取联系方式。

## 执行流程

1. 先读取或复用候选人档案、上一轮候选人分桶、用户提供的公开个人资料 / 证据。
2. 必须有候选人公开证据、项目/岗位和触达渠道；缺证据时硬停止，或转候选人档案/资料补全。只有用户明确要通用模板时才写通用模板。
3. 确认语气、长度、语言、行动请求和不可说内容；缺失但风险可控时写入假设。
4. 从公开证据中选 1-2 个个性化切入点：项目、开源贡献、论文、产品经历、社区影响或职业方向。
5. 生成 2-3 个渠道版本，保留个性化句子的证据来源，不把弱证据包装成确定事实。
6. 给出发送前人工确认清单：事实、称谓、渠道、链接、合规和用户需要补齐的签名。

## 安全边界

- 不自动发送，不点击发送按钮，不代替用户做最终确认。
- 不抓取、猜测或生成私人联系方式；没有公开职业联系方式时只建议用户已有渠道或公开资料页。
- 不编造与候选人的关系、内推关系、共同熟人、对方兴趣或求职意愿。
- 不夸大公司资源、融资、团队、客户、影响力或岗位确定性。
- 不承诺薪资、签证、入职时间、职级、title、offer 或面试结果。
- 不使用敏感 / 私人信息，包括年龄、家庭、健康、住址、私人邮箱猜测、非公开社交内容。
- 不暗示“我们一直关注你”或“看过你很多活动轨迹”等过度了解。

## 输出格式

默认直接给可人工确认的文案，不输出工作流说明或长篇策略：

```markdown
**确认**

- 只基于用户提供或公开可见证据；不补查私人联系方式，不自动发送。
- 候选人 / 项目 / 渠道 / 语气中缺失但可保守处理的点，用一句话说明假设。

**LinkedIn 私信版**

<120-180 字，低压力，1 个个性化切入点，1 个行动请求>

**Email 版**

Subject: <自然、低压、不过度销售>

<180-260 字，包含身份、公开证据切入、项目相关性、低压力行动请求、退出选项>

**发送前确认**

1. 核候选人姓名、称谓、公司和公开经历原文。
2. 补 `[公司/项目名]`、`[你的名字]`、公开职业签名或用户已有发送渠道。
3. 不加入薪资、职级、签证、入职时间、私人联系方式、自动发送或过度关注表述。
```

## 写作要求

- 开头直接给草稿，不输出“我会如何做”、使用了哪个 skill、命令或工具过程。
- 个性化只用 1-2 条公开证据，使用柔性表达：`看到你公开资料里...`、`这和我们正在做的方向有交集`。
- 不判断求职意愿；使用“如果你愿意了解 / 如果方便 / 不合适也完全没关系 / 欢迎转给合适的人”。
- 邮件签名只放用户需要补齐的公开职业签名或公司主页；不要提示去找私人邮箱。
- 只有用户明确要求策略、证据表或多版本比较时，才追加文案策略、个性化证据表或更多版本。
